# Prototype 4 — Map Data: Sources, Licensing & Production Risk

**Last updated:** 2026-06-12
**Applies to:** `prototypes/app/(proto4)/prototype4/map` + `prototypes/components/map/DubaiInteractiveMap.jsx`
**Status:** ⚠️ Prototype/demo OK — **still not production-safe** (geometry/locations are FAM-sourced).

---

## 1. TL;DR

The map's geometry was originally pulled from **FAM Properties' backend
(`fam-erp.com`)** — the same backend behind **dxbinteract.com / investmentmap.ai**.

We have since **stripped out all of FAM's value-added data** (stats, prices,
yields, ratings, curriculum, their colours and ids) and now drive the map with
**our own colours** and **our own mock project numbers**. Names + marker points +
boundary shapes remain.

What's left is **structural, not field-level**: the **boundary polygon shapes**
and the **compiled set of locations** still came from FAM's database. Removing
more fields won't fix this — only changing the **source** of the geometry/points
(to OpenStreetMap or Dubai Pulse) makes it fully production-safe.

## 3. What we already removed / replaced ✅

These no longer carry FAM's data:

- **Communities** — removed `color`, `project_count`, `unit_count`, `supply`,
  `sales_volume`, `median_price_sqft`, `rental_yield`, `capital_appreciation`,
  `location_id`, `dld_area_id`. Kept only `name` + geometry.
- **Metro** — removed station line labels; kept station `name` + a `color` for
  rendering. Lines keep only `color`.
- **Medical** — removed everything except `name`.
- **Schools** — removed `rating` + `curriculum`; kept only `name`.
- **Tile colours** → our own palette (`TILE_PALETTE` in `DubaiInteractiveMap.jsx`).
- **Project numbers** → our own mock data (`mock/prototype4/communities.js`).
- **Marker icons** → drawn on `<canvas>` (nothing copied).
- **Landmark photos** → never downloaded or used.
- We also never stored FAM's written community descriptions.

---

## 4. What is still a concern 🔴🟠

1. 🔴 **Boundary polygon geometry** (`dubai_communities.geojson`) — the 164
   community shapes are FAM's curated geometry. This is the main remaining item;
   stripping properties does not change the shapes.
2. 🟠 **Compiled location set** — *which* 164 communities, 99 metro stations,
   61 medical points, 190 schools were extracted from FAM's database. A single
   name/coordinate is a fact, but a substantial extraction of their compiled
   dataset can still carry **database-right / ToS** risk.

Names and coordinates themselves are facts (low risk individually) — the issue is
the **shapes** and the **set**, which only a source change fixes.

- **Tile colours + project numbers** — ours (palette + mock).

- **Landmark photos** — never used.
- **Basemap** — MapLibre GL + **OpenFreeMap** ("positron"), open data

## 6. Attribution requirement (keep this even after the swap)

The basemap is OpenStreetMap-based, so the map **must** display
"© OpenStreetMap contributors". Already handled by the MapLibre
`AttributionControl` in `DubaiInteractiveMap.jsx` — do not remove it.

## 7. How to make it production-safe (the swap)

The map reads **static files** from `public/data/`, so swapping the data needs
**zero code changes** — just replace the files with the same structure from a
licensed/open source. Our colours + mock numbers stay on top automatically.

### Structure the map expects
`dubai_communities.geojson` — each feature: `properties.name` / `properties['name:en']`
(label, also used to match mock project counts) + `geometry` (Polygon / MultiPolygon,
`[lng, lat]`). Tile colour and the badge number are NOT read from the file — they
come from our palette and `communities.js`.

Metro/POI files: LineString + `color` for metro lines; Point features with `name`
(+ `color` for metro stations) for stations / medical / schools.

### Recommended replacement sources
| Layer | Replace with | License |
|---|---|---|
| Communities | **Dubai Pulse / DLD** official community boundaries | Government — free signup / licence |
| Communities (free alt) | **OpenStreetMap** (Overpass) | ODbL — free, attribution (⚠️ missing developer communities like Damac Lagoons, Al Furjan) |
| Metro | **OpenStreetMap** (Dubai Metro is well mapped) | ODbL |
| Schools / Medical | OpenStreetMap or Fortune DXB's own CRM list | Factual / owned |
| Landmarks | Factual list (names + coordinates) | Facts |

### Steps
1. Obtain a licensed/open boundary + points source.
2. Convert into the structure above and write the `public/data/*.geojson` files.
3. Refresh — colours, labels, hover, mock numbers all keep working.
4. Delete `scripts/fetchDxbMap.mjs` (the fam-erp fetcher) once no longer needed.

> An OpenStreetMap-based builder was prototyped earlier (`buildLegalData.mjs`). It
> can be re-created on request; `mock/prototype4/export.geojson` (OSM base) is kept
> for that purpose.

## 8. Decision log

- **Done:** stripped all of FAM's value-added fields; map now uses our own colours
  + mock project numbers; only `name` + geometry/points remain.
- **For now (demo):** still using FAM's geometry/points so all developer-community

- **Before production:** replace the geometry + location set (🔴 / 🟠) with a
